# Unity 2021 分支策略

## 分支定位
`unity2021` 是本仓库的 Unity 2021.3 LTS 兼容分支，用于维护稳定、轻依赖、可回归的 AIBridge 版本。

`main` 继续承担功能扩展职责，包括但不限于：
- 新命令和新模块
- CLI 能力增强
- Skill 文档体系升级
- 面向更高 Unity 版本的特性探索

`unity2021` 只接收以下类型的改动：
- Unity 2021.3 编译兼容修复
- 依赖降级或可选化
- 命令链路稳定性修复
- 必要的小功能补齐
- 针对 2021.3 的文档补充

## 依赖策略
- 核心包必须保持最小依赖集。
- 非核心功能不要让 `Editor` 主 asmdef 直接引用外部可选包。
- 如果需要支持 `Timeline`、`Cinemachine`、`Splines`、`ProBuilder` 等能力，应拆为独立可选模块或条件编译模块。

## 参考实现来源
可参考 `Unity-Skills` 的以下内容：
- 命令命名方式
- 参数设计
- 功能覆盖范围
- AI skill 分类方式

不建议直接引入的内容：
- 直接把高版本依赖绑进主 asmdef
- 默认自动安装额外 Unity 包
- 把 2022.3+ 的 UI Toolkit 或版本特性直接下放到 2021 分支

## 推荐工作流
1. 新能力先在 `main` 设计并验证。
2. 评估该能力是否能在 Unity 2021.3 无痛运行。
3. 若可以，抽离高版本依赖后再移植到 `unity2021`。
4. 若不可以，在 `main` 保留，并在文档中明确标记版本边界。

## 当前维护原则
- 优先稳住通信、命令注册、参数绑定、CLI 调用和 Skill 生成。
- 功能扩展遵循“先核心、后外围；先兼容、后增强”。
- 对 `unity2021` 来说，稳定性优先级高于功能覆盖率。
